部署

部署

  1. 请参考Environment Preparation完成环境准备工作。

  2. 若Mind Studio所在Ubuntu系统未连接网络，请参考本步骤准备模型文件；若Mind Studio所在Ubuntu系统已连接网络，部署脚本会自动下载模型文件，此操作可忽略。

    请参考网络模型下载下载视频结构化应用使用的如下模型文件，并将.om文件放置到videoanalysis应用代码的根目录下，例如/home/ascend/ascenddk/smartcity/videoanalysis；若Mind Studio所在Ubuntu系统已连接网络，部署脚本会自动下载模型文件，此操作可忽略。

    • car_color.om：车辆颜色识别模型
    • car_type.om：车辆品牌识别模型
    • vgg_ssd.om：目标检测模型
    • pedestrian.om：人体特征推理模型
    • car_plate_detection.om：车牌检测模型
    • car_plate_recognition.om：车牌识别模型
    • face_detection.om：人脸检测模型
    • inception_age.om：年龄识别模型
    • inception_gender.om：性别识别模型

  3. 若Mind Studio所在Ubuntu系统未连接网络，请参考本步骤准备开源工具ffmpeg；若Mind Studio所在Ubuntu系统已连接网络，部署脚本会自动下载ffmpeg代码，此操作可忽略。

    由于当前Application使用了开源工具ffmpeg对视频文件解封，使用前需要用户在Mind Studio所在服务器中以Mind Studio安装用户下载ffmpeg 4.0分支的代码包到videoanalysis应用代码的根目录下，解压后目录名称请使用ffmpeg，例如/home/ascend/ascenddk/smartcity/videoanalysis/ffmpeg。

    说明：
    ffmpeg 4.0代码下载地址：https://github.com/FFmpeg/FFmpeg/tree/release/4.0。

  4. 以Mind Studio安装用户进入videoanalysis应用代码所在根目录，如/home/ascend/ascenddk/smartcity/videoanalysis。

  5. 执行部署脚本，进行工程环境准备，包括ascenddk公共库的编译与部署、网络模型的下载、Presenter Server服务器的配置等操作。

    bash deploy.sh host_ip model_mode

    • host_ip：对于Atlas 200 DK开发者板，即为开发者板的IP地址。对于AI加速云服务器，即为Host侧的IP地址。

    • model_mode代表模型文件的部署方式，有local与internet两种值，若不填写此参数，默认为internet。

      • local：本地部署，若Mind Studio所在Ubuntu系统未连接网络，请使用local模式，使用此模式时，需要已参考2与3完成模型文件及开源工具ffmpeg的下载。
      • internet：在线部署，若Mind Studio所在Ubuntu系统已连接网络，请使用internet模式，在线下载模型文件。

    命令示例：

    bash deploy.sh 192.168.1.2 internet

    • 当提示“Please choose one to show the presenter in browser(default: 127.0.0.1):“时，请输入在浏览器中访问Presenter Server服务所使用的IP地址（一般为访问Mind Studio的IP地址）。
    • 当提示“Please input a absolute path to storage video analysis data:“时，请输入Mind Studio中已存在的绝对路径用于存储视频解析数据，此路径Mind Studio用户需要有读写权限。

    如图1所示，请在“Current environment valid ip list“中选择通过浏览器访问Presenter Server服务使用的IP地址，并输入存储视频解析数据的路径。

    图 1 工程部署示意图

  6. 启动Presenter Server。

    执行如下命令在后台启动Video Analysis应用的Presenter Server主程序。

    python3 ../../common/presenter/server/presenter_server.py --app video_analysis &

    说明：
    “presenter_server.py“在“/ascenddk/common/presenter/server“目录下（例如：/home/ascend/ascenddk/common/presenter/server），可以在此目录下执行python3 presenter_server.py -h或者python3 presenter_server.py --help查看“presenter_server.py“的使用方法。

    如图2所示，表示presenter_server的服务启动成功。

    图 2 Presenter Server进程启动

    使用上图提示的URL登录Presenter Server（仅支持Chrome浏览器），IP地址为5中输入的IP地址，端口号默为7005，如下图所示，表示Presenter Server启动成功。

    图 3 主页显示

  7. 视频结构化应用支持解析本地视频和RTSP视频流。

    • 如果需要解析本地视频，需要将视频文件传到Host侧。

      例如将视频文件car.mp4上传到Host侧的“/home/HwHiAiUser/sample“目录下。

      详细操作步骤可参考如何上传文件到Host侧。

      说明：
      支持H264与H265格式的MP4文件，如果MP4文件需要剪辑，建议使用开源工具ffmpeg，使用其他工具剪辑的视频文件ffmpeg工具可能不支持解析。

    • 如果仅解析RTSP视频流，本步骤可跳过。

Parsing the Registration Code of the Operator Plug-In

Parsing the Registration Code of the Operator Plug-In

Parsing Operator Parameters

This function is implemented using the IOpParser::ParseParams interface.

The offline model conversion tool OMG is used to read and parse model files and weight files. The plug-in parses only parameters related to custom operators.

  1. Verify the operator validity. The operator input cannot be empty.

    const caffe::LayerParameter* layer = dynamic_cast<const caffe::LayerParameter*>(op_src);
        // #### Verify the validity of input operator parameters.
        if (nullptr == layer)
        {
            printf("Dynamic cast op_src to LayerParameter failed\n");
            return FAILED;
        }

  2. Parse the operation, axis, and coefficient parameters of the operator and save the parsing result.

    std::map<::caffe::ReductionParameter_ReductionOp, std::string> operation_map = {
            { caffe::ReductionParameter_ReductionOp_SUM, "SUM" },
            { caffe::ReductionParameter_ReductionOp_ASUM, "ASUM" },
            { caffe::ReductionParameter_ReductionOp_SUMSQ, "SUMSQ" },
            { caffe::ReductionParameter_ReductionOp_MEAN, "MEAN" },
        };
        // get op params
        const ::caffe::ReductionParameter& param = layer->reduction_param();
        // set op params to op_dest
        AddOpAttr("operation", operation_map[param.operation()], op_dest);
        AddOpAttr("axis", (int32_t)param.axis(), op_dest);
        AddOpAttr("coeff", param.coeff(), op_dest);

Describing the Output Tensor

This function is implemented using the IOpBuilder::GetOutputDesc interface.

You can infer the description of the output tensor based on the description of the input tensor.

Because the descriptions （shape, data type, and data format） of the input and output tensors are the same, you can directly copy the input tensor as the output tensor. The modified code is as follows:

    virtual Status GetOutputDesc(vector<TensorDescriptor>& v_output_desc) override
    {
        v_output_desc.push_back(op_def_->input_desc(0));

        int32_t axis = -1;
        // ### Parse the axis parameter.
        if (!GetOpAttr("axis", &axis, op_def_))
        {
            printf("GetOpAttr axis failed!\n");
        }
        // Because the Da Vinci model sets all shape values to 4d, the axis parameter needs to be fixed to point to the original 2d.
        if (axis < 0) axis -= 2;

        if (axis < 0) axis += v_output_desc[0].dim_size();

        if (axis < 0 || axis >= v_output_desc[0].dim_size(）)
        {
            printf("invalid axis:%d, dim_size:%d\n", axis, v_output_desc[0].dim_size(）);
            return PARAM_INVALID;
        }
        v_output_desc[0].set_dim(axis, 1);
        return SUCCESS;
    }

Creating an Operator Binary File

This function is implemented using the IOpBuilder::BuildTvmBinFile interface.

This interface calls the Python code （code in Reduction.py） using c++ to generate an operator file. Then, the framework compiles the operator file to the model. The modified code is as follows:

    virtual Status BuildTvmBinFile(TVMBinInfo& tvm_bin_info) override
    {
        // 1. call tvm python api to build bin files
        // 2. set path to TVMBinInfo tvm_bin_info
        PyObject *pModule = NULL;
        PyObject *pFunc = NULL;
        PyObject *pArg = NULL;
        PyObject *result = NULL;

        // ### Initialize the Python.
        Py_Initialize();

        // ### Import the topi.cce module.
        PyRun_SimpleString("import sys");
        // ### Import the directory of the current user-defined operator.
        PyRun_SimpleString("sys.path.append('/projects/ReductionOP_Demo/Reduction/operator')");
        // ### Import the Python file of the current user-defined operator.
        pModule = PyImport_ImportModule("Reduction");
        if (!pModule)
        {
            PyErr_Print();
            printf("import topi.cce failed!\n");
            return FAILED;
        }
        std::string operation;
        int32_t axis = -1;
        float coeff = 1;
        // ### Parse the operation parameter.
        if (!GetOpAttr("operation", &operation, op_def_))
        {
            // ### Add exception handling and maintenance information.
            printf("GetOpAttr operation failed!\n");
        }

        // ### Parse the axis parameter.
        if (!GetOpAttr("axis", &axis, op_def_))
        {
            printf("GetOpAttr axis failed!\n");
        }

        // Because the Da Vinci model sets all shape values to 4d, the axis parameter needs to be fixed to point to the original 2d.
        if (axis < 0) axis -= 2;

        // ### Parse the coeff parameter.
        if (!GetOpAttr("coeff", &coeff, op_def_))
        {
            printf("GetOpAttr coeff failed!\n");
        }
        // ### Parse the description of the input tensor.
        TensorDescriptor input_desc = op_def_->input_desc(0);
        // ### Parse the input shape value and check whether it is 4.
        if (input_desc.dim_size() != 4)
        {
            printf("The shape size is %d, which is not 4!", input_desc.dim_size(）);
            return FAILED;
        }
        // Call the Python interface to generate an operator, where Reduction is the operator interface name.
        pFunc = PyObject_GetAttrString(pModule, "Reduction");
        std::string kernel_name = "reduction_float16_" + 
                    std::to_string(input_desc.dim(0))+ "_"
                    + std::to_string(input_desc.dim(1))+"_"
                    + std::to_string(input_desc.dim(2))+"_"
                    + std::to_string(input_desc.dim(3));
        pArg = Py_BuildValue(
            "(i,i,i,i), s, i, s, f, s",
            input_desc.dim(0), input_desc.dim(1), input_desc.dim(2), input_desc.dim(3),"float16",
            axis, operation.c_str(), coeff, kernel_name.c_str(）);
        PyEval_CallObject(pFunc, pArg);
        // ### End the Python calling.
        Py_Finalize();

        // Set the operator file path and kernel name.
        tvm_bin_info.bin_file_path = "./kernel_meta/" + kernel_name + ".o";
        tvm_bin_info.json_file_path = "./kernel_meta/" + kernel_name + ".json";
        return SUCCESS;    
    }

Registering an Operator

You need to call the OMG interface to register the information about the custom operator so that the OMG can identify the type of the custom operator during conversion.

The registration instance is as follows:

// Register the op plugin
DOMI_REGISTER_OP("Reduction")                         // op name              
    .FrameworkType(CAFFE)                             // enum type, CAFFE, TENSORFLOW
    .OriginOpType("Reduction")                        // name of the same op in CAFFE       
    .ParserCreatorFn(PARSER_FN(ReductionOpParser))    // class name of op parser 
    .BuilderCreatorFn(BUILDER_FN(ReductionOpBuilder)) // class name of op builder
    .ImplyType(ImplyType::TVM);                       // op type, TVM
}

Building an Input Data File

Building an Input Data File

To run a single operator, you need to build input data and save it in binary format to the operator project.

This section uses DataGen.py of the DDK to generate sample data of the reduction operator. The procedure is as follows:

  1. Log in to the MindStudio server as the Mind Studio installation user.

  2. Copy the DataGen.py script in the TVM sample project in DDK to the operator project directory:

    cp $HOME/tools/che/ddk/ddk/sample/tvm/DataGen.py /projects/ReductionOP_Demo

    NOTE:
    $HOME/tools is the default installation path of the DDK.

  3. Run the DataGen.py script in the operator project directory to generate sample data.

    cd /projects/ReductionOP_Demo

    python DataGen.py

    The data files shown in Figure 1 are generated in the operator project.

    Figure 1 Generated sample data

    Table 1 describes the data files.

    Table 1 Description of data files

    Data File

    Description

    reduction_input_2_3_4_sum_axis_1.data

    Indicates the input data file in binary format of the reduction operator.

    reduction_input_2_3_4_sum_axis_1.txt

    Displays the binary files of the reduction operator in txt format for users to easily view the results.

    reduction_output_2_3_4_sum_axis_1.data

    Indicates the output data verification file in binary format of the reduction operator, which is used to verify whether the output of the operator is correct after the operator runs.

    reduction_output_2_3_4_sum_axis_1.txt

    Indicates the output data file of the reduction operator, which is available to users and has the same functions as reduction_output_2_3_4_sum_axis_1.data.

    sign_input_2_4_cce.data

    Indicates the input data file in binary format of the sign operator.

    This file is not used in this example.

    sign_input_2_4_cce.txt

    Displays the binary files of the sign operator in txt format for users to easily view the results.

    This file is not used in this example.

    sign_output_2_4_cce.data

    Indicates the output data verification file in binary format of the sign operator, which is used to verify whether the output is correct after the operator runs.

    This file is not used in this example.

    sign_output_2_4_cce.txt

    Indicates the output data file of the sign operator, which is available to users and has the same functions as sign_output_2_4_cce.data.

    This file is not used in this example.

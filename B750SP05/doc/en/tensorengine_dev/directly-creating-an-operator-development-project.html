Directly Creating an Operator Development Project

Directly Creating an Operator Development Project

If you know the unsupported operators in all models, you can create an operator development project by referring to the following steps:

  1. Choose File > New > New Project on the menu bar to create a Tensor Engine project.

    Select Tensor Engine Project, enter the project name, and select a running target of the project. ASIC indicates that the running target is the EVB board or PCIe board. Atlas DK indicates that the running target is a developer board.

    Figure 1 shows project creation.

    Figure 1 Creating a Tensor Engine project

  2. Click Next. A dialog box is displayed, as shown in Figure 2.

    Figure 2 Selecting a model file

  3. Click on the right of Model File and select the prototxt file of the Caffe model. Each layer of operators in the model is displayed in the Custom Layer area. Select the Reduction operator, as shown in Figure 3. If you want to select multiple operators, select multiple layers.

    Figure 3 Selecting a model file and operators

    NOTE:
    In this example, the model file is in the DDK and is stored in the $HOME/tools/che/ddk/ddk/sample/customop/customop_caffe_demo/model directory of the Mind Studio installation user.

    If the uploaded prototxt file requires a custom caffe.proto （the built-in caffe.proto cannot parse the current model and the storage path is $HOME/tools/che/ddk/ddk/include/inc/custom/proto/caffe/caffe.proto）, the operator fails to be parsed for the first time, as shown in the following figure.

    Figure 4 Operator parse failure for the first time

    Click Upload to upload the customized caffe.proto. The system combines the required fields into the caffe.proto.

  4. Click Create. The operator project creation is complete.

    You can view the created project in the Projects area, as shown in Figure 5.

    Figure 5 Files generated by the operator project

    In the preceding figure:

    • The python file in the operator directory corresponds to the custom operator.
    • The Reduction_plugin directory contains one .cpp file and one Makefile file, which correspond to the custom operator plug-in file and compilation makefile file, respectively.

    Reduction.py and Reduction_parser.cpp files have to be edited for developing the custom operator and plug-in.

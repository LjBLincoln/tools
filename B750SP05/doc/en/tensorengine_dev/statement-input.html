Statement Input

Pre-Processing

The reduction operator performs a reduce operation on the specified axis and its subsequent ones. Therefore, you can consider the specified axis and all its subsequent axes as one axis. In this way, the input tensor can be preprocessed, and all the axes after the specified axis are regarded as one axis. You only need to perform a reduce operation on the last axis.

The preprocessing code is as follows:

if axis < 0:
    axis = len(shape) + axis
shape = list(shape)
shape1 = shape[:axis] + [reduce(lambda x, y: x * y, shape[axis:])] # Combine the specified axis and its subsequent axes into one axis.

Define an input placeholder.

data_input = tvm.placeholder(shape1, name="data_input", dtype=inp_dtype)

In TE, the input tensor is defined using the placeholder interface. The placeholder returns a tensor object, which represents a set of input data with the known size and type. The specific content of the data can only be obtained at run time.

shape = (1024,1024)
dtype = "float16"
data = tvm.placeholder(shape, name="data", dtype=dtype)

Table 1 describes the parameters.

Table 1 Parameter description

Parameter

Description

shape

Indicates a tensor size, for example, (10), (1024,1024), or (2,3,4).

name

Indicates the input name.

dtype

Indicates the data type, for example, float16 or float32.

Tensor is an important data structure in the TE. It represents a tensor object.

Table 2 Parameters of the tensor object

Parameter

Description

shape

Indicates a tensor size, for example, (10), (1024,1024), or (2,3,4).

Data type

Indicates the data type. The value is float16 or float32.

op

Describes the computation information, which can be an input (placeholder) or a computation (compute).

For example, if TE is used to describe A + B = C, A, B, and C are tensors, and their shapes and data types are the same. A and B are input data, and their op information is a placeholder. C is the sum of A and B. Therefore, the op information of C is a computation. Generally, each computation is a minor operation, for example, add a constant to a tensor or multiply two tensors.

In TE, a computation process can be represented by a group of tensors.

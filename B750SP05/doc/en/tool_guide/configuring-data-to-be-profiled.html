Configuring Data to Be Profiled

Configuring Data to Be Profiled

  1. Double-click the *.mind file in the Mind project.

  2. On the process orchestration page that is displayed, click the Profiling icon, as shown in Figure 1.

    Figure 1 Profiling icon

  3. In the Profiling dialog box that is displayed, configure the configuration items in the Hardware, Software, and Connection areas to specify the modules whose performance data needs to be profiled and the profiling interval.

    1. Set the Hardware parameters.

      Figure 2 shows the profiling attributes that can be configured for the hardware.

      Figure 2 Hardware configuration area

      Table 1 describes the details of the Hardware parameters.

      Table 1 Hardware configuration parameters

      Parameter

      Description

      Control CPU Profiling

      Whether to enable profiling for the control CPU of the NPU. When this parameter is set to ON, the PMU event on the control CPU is profiled. Currently, the default events are 0x11 and 0x8, where 0x11 indicates the number of cycles executed by the CPU and 0x8 indicates the number of executed instructions.

      TS CPU Profiling

      Whether to enable profiling for the AI task schedule (TS) CPU. When this parameter is set to ON, the PMU event on the TS CPU is profiled. Currently, the default events are 0x11 and 0x8, where 0x11 indicates the number of cycles executed by the CPU and 0x8 indicates the number of executed instructions.

      AI CPU Profiling

      Whether to enable profiling for the AI operator CPU (different from the AI core CPU). When this parameter is set to ON, the PMU event on the AI CPU is profiled. Currently, the default events are 0x11 and 0x8, where 0x11 indicates the number of cycles executed by the CPU and 0x8 indicates the number of executed instructions.

      CPU Sampling Intervals(ms)

      PMU event profiling interval of the control CPU, TS CPU, and AI CPU. The default value is 20 (ms).

      AI Core Profiling

      Processing unit responsible for the scalar, cube, and FP AI computation. Task-based and sample-based AI core profiling is supported. Currently, the default events are 0x3, 0x8, 0x9, 0xe, 0x3a, 0x3b, 0x4a, and 0x49. 0x3 indicates the number of executed cube instructions, 0x8 indicates the number of executed vector instructions, 0x9 indicates the number of executed scalar instructions, 0xe indicates the total number of instruction cycles, 0x3a indicates the number of scalar read requests for the UB, 0x3b indicates the number of scalar write requests for the UB, 0x4a indicates the number of cube int instructions, and 0x49 indicates the number of cube FP instructions. In task-based mode, the performance data is profiled by task, while in sample-based mode, the performance data is profiled at a fixed interval.

      Peripheral Profiling

      Whether to enable profiling for the peripherals. Currently, the supported peripherals are the digital vision pre-process (DVPP) and network interface collector (NIC). The default profiling interval is 10 ms.

      LLC Capacity

      Whether to enable profiling for the capacity information of the last level cache (LLC). This switch is mutually exclusive with the LLC bandwidth switch. Only one type of LLC data can be profiled at a time. Sampling Intervals indicates the LLC profiling interval. The value range is [100, 1000]. The default value is 100 (ms).

      LLC Bandwidth

      Whether to enable profiling for the read and write bandwidths of the LLC. This switch is mutually exclusive with the LLC capacity switch. Only one type of LLC data can be profiled at a time. Sampling Intervals indicates the LLC profiling interval. The value range is [100, 1000]. The default value is 100 (ms).

      DDR Profiling

      Whether to enable profiling for the read and write bandwidths of the DDR SDRAM. The default setting is OFF.

      Master ID indicates the core whose read and write bandwidths are to be profiled. The value range is 0–7. 0–3 correspond to the core IDs of the control CPU. 4–7 correspond to the core IDs of the AI CPU.

      Sampling Intervals indicates the DDR profiling interval. The value range is [100, 1000]. The default value is 100 (ms).

      NOTE:

      • The performance monitor unit (PMU) is a hardware unit of the CPU. The CPU performance data can be read by accessing related registers.
      • DVPP is short for digital vision pre-process.
      • NIC is short for network interface controller.
      • It is recommended that the DDR and LLC sampling intervals be not greater than the time required for program execution. Otherwise, data cannot be profiled.

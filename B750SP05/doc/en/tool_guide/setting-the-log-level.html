Setting the Log Level

Setting the Log Level

Currently, you can set the log level in the following three ways:

  • Set the log level on the Mind Studio page.

    In the Log window on the Mind Studio page, click in Log List. In the Config Log Level dialog box that is displayed, set the log level and click OK.

    The log level configuration in the System area applies globally. The log level configuration in the ModuleList area applies by module. If the global log level is set, the module-level log level cannot be set. To set the module-level log level, disable the log level setting in the System area, as shown in Figure 1.

    Figure 1 Disabling the global log level setting

  • Run the command on the server to set the log level.

    Log in to the server as the Mind Studio installation user, run commands export LD_LIBRARY_PATH=~/tools/che/ddk/ddk/uihost/lib and export PATH=$PATH:~/tools/che/ddk/ddk/uihost/bin in sequence to set environment variables, and run the global or module-level log level setting command.

    You can also log in to the host server as the HwHiAiUser user and run the global or module-level log level setting command. (Only the PCIe scenario is supported.)

    • To set the global log level, run the following command:

      IDE-daemon-client --IP address of the host server:Port number --log 'SetLogLevel(0)[level]'

      level can be set to error, info, warning, or debug.

    • To set the level of a specific module, run the following command:

      IDE-daemon-client --IP address of the host server:Port number --log 'SetLogLevel(1)[moduleName:level]'

      level can be set to error, info, warning, or debug.

      moduleName can be set to a module, such as slog, dlog, or cce.

    • To set the event log level, run the following command:

      IDE-daemon-client --IP address of the host server:Port number --log 'SetLogLevel(2)[enable/disable]'

      enable: Enables the event log level setting.

      disable: Disables the event log level setting.

    NOTE:
    ~/tools is the default toolpath setting, which can be customized during Mind Studio installation. You can view the value of toolpath in the scripts/env.conf file. You can run the find / -name 'env.conf' command to view the location of the env.conf file in the script directory.

  • Modify the configuration file on the server to set the slogd and dlogd log levels.

    For details about the configuration file, see Log Configuration.

    After modifying the configuration file, log in to the host server as the HwHiAiUser user, log in to the device server using SSH, and restart the slogd, dlogd, and IDE-daemon-host processes.

    • Log in to the host server, go to the /usr/local/HiAi/driver/tools directory, and restart the slogd process by running the pkill slogd;./slogd & command.
    • Log in to the host server, go to the /usr/local/HiAi/driver/tools directory, and restart the IDE-daemon-host process by running the pkill IDE-daemon; ./IDE-daemon-host & command.
    • Log in to the device server and restart the slogd process by running the pkill slogd; /var/slogd >/dev/null & command.
    • Log in to the device server and restart the dlogd process running the pkill dlogd; /var/dlogd >/dev/null & command.

    NOTE:
    For the developer board, log in to the server and restart a process by running the pkill Process name; /var/Process name >/dev/null & command .

Overview

Overview

Mind Studio provides an efficient, easy-to-use, and flexible tool Profiling targeting the multi-node and multi-module heterogeneous system on the host and device. Currently, the tool supports multi-card and multi-chip scenarios. This tool quickly identifies key performance bottlenecks and provides suggestions on performance optimization, hence ensures the ultimate performance of your product.

Mind Studio can collect, analyze, and display the performance data of hardware and software in graphical or command line mode. The overall process is as follows:

  1. Configure the data to be profiled, including the hardware and software performance data.

    The hardware performance data includes the performance monitor unit (PMU) events on the control CPU, PMU events on the task schedule CPU, PMU events on the AI CPU, and performance data of peripheral devices.

    The software performance data refers to the performance data of modules such as HiAI Engine, the offline model inference engine (OME), and the runtime and task schedule (RTS).

  2. Profile the performance data.

    Configure the connection between Mind Studio and the host before profiling the performance data.

  3. View the performance analysis results.

    On the Mind Studio GUI, performance analysis results are displayed in the following three dimensions: Summary, Timeline, and functions (including AI CPU Function and Control CPU Function).

    NOTE:
    Before running the Profiling tool, ensure that:

    1. The non-root user folder in the environment where Profiling runs has the 750 permission.
    2. The host connected to the device has the permission to run a compiled instance, that is:
    3. Permission to access and execute the dependent library: Access the path where the dependent library is located as the current user and query whether the dependent library can be executed by the current user.
    4. Permission to check the compiled instance: After running the process orchestration instance, switch to the project path in ~/HIAI_PROJECTS/workspace_mind_studioto check whether the instance can be executed.

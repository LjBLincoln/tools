Summary

The Summary tab displays the collected performance data in tables, covering the following aspects:

You can click next to a table to export the performance data to an Excel file on the local PC.

  1. Collection Info area: Displays the data collection start time and end time and the size of the collected performance data, as shown in Figure 1.

    Figure 1 Collection Info area

  2. Host Info area: Displays the OS and CPU information about the host where the performance data is collected. For the developer kit form, the host and device are both located on the miniRC. Therefore, the collected data is the basic information about the miniRC, as shown in Figure 2.

    Figure 2 Host Info area

  3. Device area:

    • Displays the information about the device where the performance data is collected, as shown in Figure 3.

      Figure 3 Device area

    • HiAI Engine area: Displays the start time and end time of each engine, as shown in Figure 4.

      Figure 4 HiAI Engine area

      NOTE:
      In the HiAI Engine data, a graph name is in the format graph+device/host+graphID, and an engine name is in the format Engine name(thread ID).

    • OME area: Displays the time consumed by data input to, inference of, and data output from the model, and details about the running of each operator, including the operator name, operator type, running time, and memory usage, as shown in Figure 5.

      Figure 5 OME area

      NOTE:
      For the development of the HiAI Engine code, if a thread for invoking the inference engine is created in the code, when Profiling is executed to collect OME data after the project is compiled and executed, the internal thread of the system cannot match the engine. As a result, the Model Statistic table in the profiling results cannot display the Engine Name information.

    • RTS areas: Display the runtime API invocation details and task scheduling information of the Task Scheduler, as shown in Figure 6.

      Figure 6 RTS area

    • LLC area under NPU: Displays the read and write bandwidths and hit rate of the LLC of all cores, as shown in Figure 7.

      Figure 7 LLC area under NPU

    • DDR area: Displays the DDR information, as shown in Figure 8.

      Figure 8 DDR area

    • Control CPU areas: Display the collected PMU events and hotspot functions of the control CPU, as shown in Figure 9.

      Figure 9 Control CPU areas

    • Control CPU Usage area: Display the CPU usage, as shown in Figure 10.

      If the collection time is less than 1s, the diagram is not displayed.

      Figure 10 Control CPU Usage area

    • LLC area under Control CPU: Displays the total used capacity of the LLC of the control CPU, as shown in Figure 11.

      Figure 11 LLC area under Control CPU

    • TS CPU area: Displays the TS CPU usage, as shown in Figure 12.

      If the collection time is less than 1s, the diagram is not displayed.

      Figure 12 TS CPU area

    • AI CPU areas: Display the collected PMU events and hotspot functions of the control CPU, as shown in Figure 13.

      Figure 13 AI CPU area

    • AI CPU Usage area: Displays the AI CPU usage, as shown in Figure 14.

      If the collection time is less than 1s, the diagram is not displayed.

      Figure 14 AI CPU Usage area

    • LLC area under AI CPU: Displays the total used capacity of the LLC of the AI CPU, as shown in Figure 15.

      Figure 15 LLC area under AI CPU

    • AI Core area: Displays the metrics data calculated based on the AI core PMU events, as well as the AI core status information, as shown in Figure 16.

      Figure 16 AI Core area

      Note the following:

      NOTE:
      Currently, the default events are 0x3, 0x8, 0x9, 0xe, 0x3a, 0x3b, 0x4a, and 0x49. 0x3 indicates the number of executed cube instructions, 0x8 indicates the number of executed vector instructions, 0x9 indicates the number of executed scalar instructions, 0xe indicates the total number of instruction cycles, 0x3a indicates the number of scalar read requests for the UB, 0x3b indicates the number of scalar write requests for the UB, 0x4a indicates the number of cube int instructions, and 0x49 indicates the number of cube FP instructions.
      The formulas for calculating the AI Core metrics are as follows. The sum of the following ratios is not 1 because there is an inclusion relationship among them.

      1. total_time reads the number of task cycles from the PMU_TASK_CYC_CNT register. Then calculate the time based on the operating frequency.
      2. mac_fp16_ratio indicates the ratio of the cycles of cube FP instructions to the cycles of all instructions, which can be obtained by using the formula: mac_fp16_ratio = 1.0 x SUM(0x49)/SUM(PMU_TASK_CYC_CNT)
      3. mac_int8_ratio indicates the ratio of the cycles of cube INT instructions to the cycles of all instructions, which can be obtained by using the formula: mac_int8_ratio = 1.0 x SUM(0x4a)/SUM(PMU_TASK_CYC_CNT).
      4. vec_ratio indicates the ratio of VEC instructions to all instructions, which can be obtained by using the formula: vec_ratio = 1.0 x SUM(0x8)/SUM(PMU_TASK_CYC_CNT)
      5. mac_ratio indicates the occupancy ratio of the cube command (excluding the write request of the special register) in all instructions, which can be obtained by using the formula: mac_ratio = 1.0 x SUM(0x3)/SUM(PMU_TASK_CYC_CNT)
      6. vec_ratio indicates the ratio of scalar instructions to all instructions, which can be obtained by using the formula: scalar_ratio = 1.0 x SUM(0x9)/SUM(PMU_TASK_CYC_CNT)
      7. scalar_ld_ratio indicates the ratio of the number of read instructions requesting to read the UB to all instructions, which can be obtained by using the formula: scalar_ld_ratio = 1.0 x SUM(0x3a)/SUM(PMU_TASK_CYC_CNT).
      8. scalar_st_ratio indicates the ratio of the number of read instructions requesting to write the UB to all instructions, which can be obtained by using the formula: scalar_st_ratio = 1.0 x SUM(0x3b)/SUM(PMU_TASK_CYC_CNT).

    • Peripherals area: Displays the task time, number of frames and usage of each DVPP engine, and network TX and RX status of the NIC, as shown in Figure 17.

      Figure 17 Peripherals area

      NOTE:
      A device in the PCIe card form does not have NIC data. Therefore, NIC profiling should be disabled.

    • Time Consumption Statistics area: Displays the uplink and downlink data of each AI core task, as shown in Figure 18.

      Figure 18 Uplink and downlink data

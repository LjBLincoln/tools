Local Preparation

Local Preparation

Preparing the Environment

  • The Linux server （server where Mind Studio is located） connected to the SD card in USB mode is connected to the network.

  • Before preparing the SD card, ensure that the qemu-user-static, binfmt-support, YAML, and cross compiler have been installed on the Linux server where Mind Studio is located.

    If the Linux server has been connected to the network, run the following commands to install them:

    su - root

    apt-get install qemu-user-static binfmt-support python3-yaml gcc-aarch64-linux-gnu g++-aarch64-linux-gnu

    If the Linux server is not connected to the network, download the sources yourself and install them.

Obtaining Software Packages

  • Obtain the SD card preparation scripts make_sd_card.py, make_ubuntu_sd.sh, the Mini package, and the Ubuntu package.

    Table 1 shows the download information.

    Table 1 Software packages

    Package

    Name

    Description

    Entry script for preparing the SD card

    make_sd_card.py

    Obtain it from the tools repository at https://github.com/Ascend.

    Script for preparing the SD card OS

    make_ubuntu_sd.sh

    Obtain it from the tools repository at https://github.com/Ascend.

    Mini package

    mini_developerkit-xxx.rar

    The corresponding software integrity verification file is mini_developerkit-xxx.rar.asc.

    Software package of the developer board

    Obtain the software from the ascenddk-private repository in https://github.com/Ascend.

    Verify the software integrity of the downloaded software package by referring to Verifying Software Package Integrity.

    Ubuntu package

    ubuntu-xxx-server-arm64.iso

    NOTE:

    xxx indicates the version number.

    On Linux, the Ubuntu version must be 16.04.3.

    Download the software of the required version from http://old-releases.ubuntu.com/releases/16.04.3/ and install the software.

    NOTE:
    • Ubuntu must be a server-arm64 version.
    • The Ubuntu version must be the same as the Ubuntu version where Mind Studio is installed.

    NOTE:
    After the software is downloaded, keep their names unchanged.
    If you want to download a single file from GitHub, right-click Raw and choose Save link as from the shortcut menu.

Procedure

  1. Upload make_sd_card.py, make_ubuntu_sd.sh, the Mini package, and the Ubuntu package obtained from Obtaining Software Packages to any directory on the server where Mind Studio is located, for example, /home/ascend/mksd.

    NOTE:
    The preceding scripts and software packages must be stored in the same directory.
    Only the software package of one version can be stored in the current directory.

  2. Switch to the root user and go to the /home/ascend/mksd directory where the card creation script is stored.

    su - root

    cd /home/ascend/mksd/

  3. （Optional） Modify NETWORK_CARD_DEFAULT_IP and USB_CARD_DEFAULT_IP in the make_sd_card.py file to change the IP address of developer board. The default IP address of the developer board is 192.168.1.2 in USB connection mode and is 192.168.0.2 in network cable connection mode.

  4. Run the make_sd_card.py script.

    Run the fdisk -l command to query the name of the USB device where the SD card is located, for example, /dev/sda.

    Run the make_sd_card.py script:

    python3 make_sd_card.py local /dev/sda

    • local indicates that the SD card is prepared locally.
    • /dev/sda is the name of the USB device where the SD card is located.

    If Figure 1 is displayed, the card is successfully prepared.

    Figure 1 Message indicating successful SD card preparation

    NOTE:
    If the card preparation fails, check the log files in the sd_card_making_log folder in the current directory.

  5. （Optional） Remove the jumper.

    NOTE:
    If no card reader is available, remove the jumper cap after the SD card is prepared.

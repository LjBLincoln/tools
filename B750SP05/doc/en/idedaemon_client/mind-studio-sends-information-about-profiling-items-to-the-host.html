Mind Studio Sends Information About Profiling Items to the Host

Mind Studio Sends Information About Profiling Items to the Host

Command Function

Allows Mind Studio to send the information about profiling items to the host and starts the profiling.

Command Format

IDE-daemon-client --host <host_ip>:<port> --profile "<params>"

Parameter Description

Table 1 Parameter description

Parameter

Description

Mandatory

Default Value

--host
  • host_ip: Specifies the IP address of the host.
  • port: Specifies the port number.

Yes

The default value of port is 22118.

--profile

Specifies profiling items in params.

Yes

-

Prerequisites

Project files compiled on Mind Studio are copied to the corresponding directories on the host in advance.

For example, if the project name on Mind Studio is test and compiled project files are stored in the /projects/test directory, copy the /projects/test directory and the files in it to the directory (for example, ~/HIAI_PROJECTS/workspace_mind_studio) on the host.

Example

  1. Log in to the Mind Studio server as the Mind Studio installation user.

  2. Run the following commands to set the environment variables:

    export  LD_LIBRARY_PATH=~/tools/che/ddk/ddk/uihost/lib
export  PATH=$PATH:~/tools/che/ddk/ddk/uihost/bin

  3. Run the following command:

    IDE-daemon-client --host xx.xx.xx.xx:22118 --profile "{\"ddk_dir\": \"~/tools/che/ddk/ddk\",\"app\": \"main\",\"app_dir\": \"/xxx/xxx\", \"umode\": \"MIND\",\"result_dir\": \"/projects/test/profiling_output\",\"peripheral_profiling\": \"\",\"ts_cpu_profiling\": \"off\",\"ai_cpu_profiling\": \"off\",\"rts_profiling\": \"on\",\"ai_core_profiling_mode\": \"\",\"ai_core_profiling\": \"off\",\"hiai_engine_profiling\": \"off\",\"framework_profiling\": \"off\",\"ctrl_cpu_profiling\": \"off\",\"profiling_mode\": \"online\",\"llc_profiling\": \"off\",\"ddr_profiling\": \"on\"}"

    NOTE:

    • Replace xx.xx.xx.xx after --host with the actual IP address of the host.
    • Replace /xxx/xxx after app_dir with the directory of the app on the host. For example, if the files related to the compiled project on Mind Studio are copied to the ~/HIAI_PROJECTS/workspace_mind_studio directory on the host, the app directory is ~/HIAI_PROJECTS/workspace_mind_studio/project name/user-defined directory.
    • Replace app with the application name (for example, main) in the directory specified by app_dir.
    • Replace result_dir with the actual directory on the Mind Studio server. After running the command, you can view the profiling result in the directory specified by result_dir.
    • For details about how to configure profiling items, see section Profiling in the Ascend 310 Mind Studio Auxiliary Tools.

JPEGD

JPEGD

The JPEGD module is used to decode images in .jpg format.

  • The JPEGD supports the following image formats:

    The JPEG (color space: YUV, subsample: 444/422/420/400) supports only Huffman encoding, does not support arithmetic encoding, does not support progressive encoding, and does not support the JPEG 2000 format.

  • The JPEGD supports the following resolutions:

    Maximum resolution: 8192 x 8192

    Minimum resolution: 32 x 32

  • The JPEGD supports the following output formats:

    JPEG(444) -> YUV444 / YUV420 semi-planar with V before U

    JPEG(422) -> YUV422 / YUV420 semi-planar with V before U

    JPEG(420) -> YUV420 / YUV420 semi-planar with V before U

    JPEG(400) -> YUV420/ UV data is padded using 0 x 80.

  • Memory limit:

    • The start address of the cache must be 128-byte aligned.
    • The virtual addresses of the input and output caches for the same task should be in the same 4 GB space.

The performance specifications of the JPEGD are as follows:

NOTICE:
The programming specification of the JPEGD requires the use of the camel-case style. The input parameters and output parameters in the original kernel style are supported, and new camel-case APIs are provided for the caller. Therefore, the JPEGD supports two sets of APIs.

Input Parameter: JpegdIn

The API is updated in the B590 version.

Member Variable

Description

unsigned char* jpegData

Data of the input .jpg image. The start address is 128-byte aligned and the memory is allocated using the 2 MB huge page table. The virtual addresses of the input and output caches for the same task must be in the same 4 GB space.

uint32_t jpegDataSize

Length of the input buffer. According to the hardware restriction, the length is 8 bytes longer than the actual stream length.

bool IsYUV420Need

Whether data is output in YUV420 semi-planar format. The options are true and false. The JPEGD supports image output in raw format and in YUV420 semi-planar format. The YUV420 of grayscale image output is in fake420 format.

bool isVBeforeU

V is before U in the JPGED output. This field is reserved.

Output Parameter: JpegdOut

The API is updated in the B590 version.

Member Variable

Description

unsigned char* yuvData

Buffer for the output YUV data. The width and height of the image are aligned values.

uint32_t yuvDataSize

Length of the output YUV data. The data length is calculated based on the aligned width and height.

uint32_t imgWidth

Width of the output YUV image

uint32_t imgHeight

Height of the output YUV image

uint32_t imgWidthAligned

Aligned width of the output image. The width is 128-pixel aligned.

uint32_t imgHeightAligned

Aligned height of the output image. The width is 16-pixel aligned.

JpegCalBackFree cbFree

Callback function for releasing the output buffer. For details, see Calling Example.

Input Parameter: jpegd_raw_data_info

Member Variable

Description

Data of the input JPG image. The start address is 128-byte aligned, and the 2 MB huge page table is used for allocation.

Length of the input buffer. According to the hardware restriction, the length is 8 bytes longer than the actual stream length.

jpegd_raw_format in_format

YUV sampling format of the input image, which does not need to be padded. Just use the default value.

Whether data is output in YUV420 semi-planar format. The JPEGD supports image output in raw format and in YUV420 semi-planar format. The YUV420 of grayscale image output is in fake420 format.

bool isVBeforeU

Only true is supported, that is, V is before U in the JPGED output. This field is reserved.

Calling Example

  • Example of the callback function:

    /**
* The application and release of the output buffer for hardware decoding (encoding) and software decoding (encoding) are inconsistent. The function objects are encapsulated
* to shield the release details of the output buffer, and the binding function is required to process
* binary and one-variable functions as parameterless functions, which can be used as callback functions to be called by the caller.
* usage: outBuf.cbFree = JpegCalBackFree( .. )
*        outBuf.cbFree()
*/
class JpegCalBackFree
{
 public:
 JpegCalBackFree( )
 : isMunmapUse_( false ), addr4Free_( NULL ), siz4Free_( 0 ) {}
 void setBuf( void* addr4Free )
 {
  isMunmapUse_ = false;
  addr4Free_   = addr4Free;
 }
 void setBuf( void* addr4Free, size_t siz4Free )
 {
  isMunmapUse_ = true;
  addr4Free_   = addr4Free;
  siz4Free_    = siz4Free;
 }
 void operator() ()
 {
  // if isMunmapUse_ is true, use munmap() to free the buf
  if( isMunmapUse_  && NULL != addr4Free_ )
  {
   if( -1 == munmap( addr4Free_, ALIGN_UP( siz4Free_ , MAP_2M ) ) )
   std::cerr << "free buf with munmap error" << std::endl;
  }
  // if isMunmapUse_ is false, use free() to free the buf
  if( !isMunmapUse_ && NULL != addr4Free_ )  free( addr4Free_ );
  addr4Free_ = NULL;  // safe free
 }
 private:
 // no copyable to avoid double free, private copy constructor and =
 JpegCalBackFree( const JpegCalBackFree& );
 const JpegCalBackFree& operator= ( const JpegCalBackFree& );
 private:
 bool   isMunmapUse_;
 void* addr4Free_;
 size_t siz4Free_;
};

  do{ // for resource inBuf

   fread( jpegdInData.jpeg_data, 1, fileLen, fpIn );
                       // dvppCtl input data
   dvppapi_ctl_msg dvppApiCtlMsg;

    DestroyDvppApi(pidvppapi);
    break;
    }
                               // Release the DVPP API handle.
    DestroyDvppApi(pidvppapi);
   }
   else
   {
    API_LOGE("can not create dvpp api\n");
    break;
   }
   char outFl[56] = "";
   snprintf(outFl, sizeof(outFl), "dvppapi_jpegd_w%d_h%d_%d.yuv",
   jpegdOutData.img_width_aligned,
   jpegdOutData.img_height_aligned,
   jpegdOutData.out_format);
   FILE* fpOut = fopen( outFl, "wb" );
   if( fpOut )
   {
    fwrite( jpegdOutData.yuv_data, 1, jpegdOutData.yuv_data_size, fpOut );
    fflush( fpOut );
    fclose( fpOut );
   }
   else API_LOGE("can not open output file %s \n", outFl );
                       jpegdOutData.cbFree(); // Release output buffer.

 }while(0); // for resource fpIn

rawdata_collage_config

rawdata_collage_config

Member Variable

Description

Value Range

Int: image_type

Image type. The image type must be YUV420_semi_planar.

0

(yuv420_semi_planar)

Int: image_rank_type

Image format (NV12 or NV21). The image formats of the four images to be stitched must be the same.

The value is 0 or 1.

0 corresponds to NV12, while 1 corresponds to NV21.

Int: bit_width

Bit depth

The default value is 8. You do not have to set this parameter.

Int: in_width

Image width

The value must be an even number in the range of [32, 4096].

Int: in_high

Image height

The value must be an even number in the range of [32, 4096].

Int: width_stride

Width stride of an image

Stride of the image data in the buffer. If the image size is not aligned in the buffer, the width stride of a picture-based image is the same as the width of the picture-based image. Otherwise, set this parameter to a value based on an aligned image size.

Int: high_stride

Height stride of an image

Stride of the image data in the buffer. If the image size is not aligned in the buffer, the height stride of a picture-based image is the same as the height of the picture-based image. Otherwise, set this parameter to a value based on an aligned image size.

int collage_type

Image format, for example, 1 x 4 (one row and four columns), 4 x 1 (four rows and one column), and 2 x 2 (default, two rows and two columns)

The value is 0, 1, or 2.

0 corresponds to two rows and two columns (default). 1 corresponds to one row and four columns. 2 corresponds to four rows and one column.

char *: in_buffer[collage_num]

Input buffer for the four images used for stitching

Ensure that the size of the input buffer of a picture-based image is the same as its width and height after aligned by width_stride and high_stride.

For a picture-based image in YUV420 semi-planar NV12 format, the image size is (Width after alignment) x (Height after alignment) x 1.5. If the input buffer size is inconsistent with this calculation result, the invocation fails.

shared_ptr<AutoBuffer> auto_out_buffer

Output address of the stitched image

This parameter is a smart pointer transferred into the DVPP, used for the caller to allocate the output buffer.

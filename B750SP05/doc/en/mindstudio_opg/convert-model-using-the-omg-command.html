Convert Model Using the omg Command

Convert Model Using the omg Command

Function

Converts the offline model.

Encryption and non-encryption scenarios are supported. However, non-encryption scenarios are not recommended because there are risks. You are advised to use the encryption mode. If the non-encryption mode is used, the framework does not verify the integrity of the model. This integrity is verified and guaranteed by the user when the user deploys the application.

Instructions

You can run the omg command in either of the following ways:

  • Log in to the server on the host as the HwHiAiUser user, and run the omg command. This section uses this method as an example.

  • Log in to the Mind Studio server as the Mind Studio installation user, switch to the ~/tools/che/ddk/ddk/uihost/bin directory, configure environment variables, and run the omg command.

    Configure environment variables. Especially, replace /home/ascend/tools with the actual tool path.

    export LD_LIBRARY_PATH=/home/ascend/tools/che/ddk/ddk/uihost/lib/

    The following is an example of the omg command:

    ./omg --h

    NOTE:
    ~/tools is the default toolpath setting, which can be customized during Mind Studio installation. You can view the value of toolpath in the scripts/env.conf file. You can run the find / -name 'env.conf' command to view the location of the env.conf file in the script directory.

Usage Example

  1. Log in to the server on the host as the HwHiAiUser user.

  2. Run the following commands to generate an encrypted model file:

    omg --model=/usr/local/HiAI/driver/tools/test/resnet18.prototxt --weight=/usr/local/HiAI/driver/tools/test/resnet18.caffemodel --framework=0 --output=/usr/local/HiAI/driver/tools/test/out/caffe_resnet18 --encrypt_mode=0 --encrypt_key=/usr/local/HiAI/driver/tools/test/ek_key --hardware_key=/usr/local/HiAI/driver/tools/test/isv_hw_key --certificate=/usr/local/HiAI/driver/tools/test/isv_cert.crt --private_key=/usr/local/HiAI/driver/tools/test/isv_prikey.key

    After the commands are executed successfully, you can view the encrypted model file （for example, caffe_resnet18.om） and the encrypted file for decrypting the model file (for example, caffe_resnet18.PASSCODE) in the path specified by the output parameter.

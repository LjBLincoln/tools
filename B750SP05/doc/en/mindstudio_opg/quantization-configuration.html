Quantization Configuration

Quantization Configuration

With Quantization enabled during offline model conversion （for details, see 5.2.1.1 "Adding a Custom Model Component"）, after the model conversion is complete, you can view the parameter configurations in the convertModel.log file of the project.

Format of the Configuration File

  • The weight, offset, and data quantization of convolution and fully connected operators with or without offset are supported. The scalar weight and vector weight are supported.

  • The multiple network input scenario is supported. Images （IMAGE） and binary files (BINARY) are supported.

    • Images （IMAGE） can be in .jpg, .png, or .bmp format.

    • Table 1 describes the supported formats of binary files （BINARY）.

      Table 1 Binary file formats

      Offset

      Type

      Value

      Description

      0000

      32bit int

      magic

      magic number

      0004

      32bit int

      50

      input num

      0008

      32bit int

      3

      input channels

      0012

      32bit int

      28

      input height

      0016

      32bit int

      28

      input width

      ...

      float

      126

      pixel

      NOTE:
      The header of binary file contains 20 bytes with five int numbers. The first is 510 （magic_num）, which is used for verification. The other four numbers are n, c, h, and w. Data following them are of the float type. The data records are equal to the value of (n*c*h*w). For non-4D data, the data needs to be supplemented to four dimensions and the value of the supplemented dimension is 1.

Templates of the Configuration File

The name of the quantization configuration file is user-defined. For details about the file content, see the following templates:

  • Single-input scenario （IMAGE）

    device:USE_CPU
quantize_algo:HALF_OFFSET
weight_type:VECTOR_TYPE
preprocess_parameter:
{
input_type:IMAGE
image_format:BGR
input_file_path:'calibration/image_set'
mean_value:104.0
mean_value:117.0
mean_value:123.0
standard_deviation:1.0
}

  • Multi-input scenario （IMAGE+BINARY）

    device:USE_CPU
quantize_algo:HALF_OFFSET
weight_type:VECTOR_TYPE
preprocess_parameter:
{
input_type:IMAGE
image_format:BGR
input_file_path:'calibration/image_set'
mean_value:104.0
mean_value:117.0
mean_value:123.0
standard_deviation:1.0
}
preprocess_parameter:
{
input_type:BINARY 
input_file_path:'calibration/img_info.bin'
}

  • Multi-input scenario （IMAGE+BINARY）, including all parameters

    device:USE_CPU
quantize_algo:HALF_OFFSET
weight_type:VECTOR_TYPE
bin:150
type:KL
inference_with_data_quantized:false
inference_with_weight_quantized:true
super_parameter:
{
min_percentile:PERCENTILE_HIGH
max_percentile:PERCENTILE_MID
start_ratio:0.7
end_ratio:1.3
step_ratio:0.01
}
exclude_op:'fc1000'
batch_count:50
preprocess_parameter:
{
input_type:IMAGE
image_format:BGR
input_file_path:'calibration/image_set'
mean_value:104.0
mean_value:117.0
mean_value:123.0
standard_deviation:1.0
}
preprocess_parameter:
{
input_type:BINARY 
input_file_path:'calibration/img_info.bin'
}

NOTE:
When using this configuration file, set the required parameters as required. The parameters in the preceding templates are only recommended settings. Generally, the recommended settings do not need to be modified （except the input file setting）.

Parameters in the Configuration File

The global setting parameters are described as follows.

Table 2 device

Name

Inference mode

Type

enum

Value Range

USE_CPU

Description

USE_CPU: Uses the CPU for inference.

Note

Currently, the CPU is used for quantization inference.

Recommended Setting

USE_CPU

Table 3 quantize_algo

Name

Quantization mode

Type

enum

Value Range

NON_OFFSET/HALF_OFFSET

Description

There are two quantization mapping formulas:

  • With offset: q_uint8 = round(d_float/scale) – offset
  • Without offset: q_int8 = round(d_float/scale)

The weight and data are involved in the mapping.

  • NON_OFFSET: The weight and data use the without-offset mode.
  • HALF_OFFSET: The data uses the with-offset mode, and the weight uses the without-offset mode.

Note

Both NON_OFFSET and HALF_OFFSET modes are supported.

Recommended Setting

HALF_OFFSET

Table 4 weight_type

Name

Weight quantization mode

Type

enum

Value Range

VECTOR_TYPE/SCALAR_TYPE

Description

For a convolution operator, it may have multiple convolution cores, and the corresponding quantization parameters may be different.

  • VECTOR_TYPE: Each convolution core uses a separate group of quantization parameters.
  • SCALAR_TYPE: Multiple convolution cores use the same group of quantization parameters.

Note

VECTOR_TYPE is recommended.

Recommended Setting

VECTOR_TYPE

Table 5 preprocess_parameter

Name

Preprocessing and input parameters

Type

Struct

Value Range

Including input_type, image_format, input_file_path, mean_value, and standard_deviation

Description

Preprocessing and input settings

Note

The number of preprocess_parameter parameters must be the same as the number of operators input to the network.

Recommended Setting

None

Table 6 input_type

Name

Input data format

Type

enum

Value Range

IMAGE/BINARY

Description

Input type

  • IMAGE: image format
  • BINARY: binary format

Note

Configure the value as required.

Recommended Setting

None

Table 7 image_format

Name

Order of the three image input channels

Type

enum

Value Range

BGR/RGB

Description

Order of the three image input channels during model training

Note

This parameter needs to be set for IMAGE input. This parameter is determined based on the order of the input channels during network training. Generally, the order is BGR during network training.

Recommended Setting

BGR

Table 8 input_file_path

Name

Address of the input data

Type

string

Value Range

The path cannot contain Chinese characters, special characters, or spaces.

Description

Folder or file to be quantified

Note

Set this parameter based on the actual network configuration.

  • For IMAGE input, specify the folder.
  • For BINARY input, specify the file.

Recommended Setting

Set this parameter to the image folder or binary file as required.

Table 9 mean_value

Name

Mean value for image preprocessing

Type

float

Value Range

[0, 255.0]

Description

Mean value of a channel for image preprocessing

Note

This parameter needs to be set for IMAGE input. The count of mean values is determined by the channel count. Three mean values need to be configured for the RGB channels as follows:

mean_value: 104.0

mean_value: 117.0

mean_value: 123.0

Recommended Setting

None

Table 10 standard_deviation

Name

Standard deviation for image preprocessing

Type

float

Value Range

[0, FLOAT_MAX]

Description

Standard deviation for image preprocessing

Note

This parameter needs to be set for IMAGE input. One standard deviation is used for multiple channels.

If the input range is beyond the range that can be represented by the float type, the quantization precision of the model cannot be guaranteed.

If 0 ≤ standard_deviation ≤ 0.00001, the value of standard_deviation is 1.0.

Recommended Setting

1.0: The image value range (Original image value – Mean value) does not change. 255.0: If the image value range is [0, 255], the value can be condensed to [0, 1.0].)

Table 11 bin

Name

Data range of the mapping histogram

Type

uint32

Value Range

[0, 1000]

Description

Data range of the histogram statistics

Note

The statistics histogram is required during the calculation of scatter. The value of this parameter determines the maximum value of histogram statistics. If this parameter is not set or set to 0, the default value 150 is used.

Recommended Setting

100/150/200/250

Table 12 type

Name

Type of the divergence calculation indicator

Type

enum

Value Range

KL/SYMKL/JSD

Description

KL: Kullback-Leibler Divergence; SYMKL: Symmetric Kullback-Leibler Divergence; JSD: Jensen-Shannon Divergence

Note

Different divergence types correspond to different calculation methods. The default value is KL.

Recommended Setting

None

Table 13 inference_with_data_quantized

Name

Whether to use the quantized data as the inference input

Type

bool

Value Range

true/false

Description

Whether to use the quantized data as the inference input

Note

When this parameter is set to true, the quantization of the input data is simulated. The default value is false.

Recommended Setting

false

Table 14 inference_with_weight_quantized

Name

Whether to use the quantized data as the inference input

Type

bool

Value Range

true/false

Description

Whether to use the quantized data as the inference input

Note

When this function is enabled, the quantization and de-quantization of the weight data are simulated. The default value is true.

Recommended Setting

true

Table 15 super_parameter

Name

Searches parameters.

Type

Struct

Value Range

Including min_percentile, max_percentile, start_ratio, end_ratio, and step_ratio

Description

Searches parameters.

Note

You are advised to retain the default value.

Recommended Setting

None

Table 16 min_percentile

Name

Position for searching for the minimum number

Type

enum

Value Range

PERCENTILE_HIGH/PERCENTILE_MID/PERCENTILE_LOW

Description

Determines the minimum number to be used. For example, if there are 100 numbers, the value 1.0 indicates that number 0 (100 – 100 x 1.0) is used as the minimum number, that is, the first smallest number.

Note

PERCENTILE_HIGH: 1.0

PERCENTILE_MID: 0.99999

PERCENTILE_LOW: 0.9999

Recommended Setting

PERCENTILE_HIGH

Table 17 max_percentile

Name

Position for searching for the maximum number

Type

enum

Value Range

PERCENTILE_HIGH/PERCENTILE_MID/PERCENTILE_LOW

Description

Determines the maximum number to be used. For example, if there are 100 numbers, the value 1.0 indicates that number 0 (100 – 100 x 1.0) is used as the maximum number, that is, the first largest number.

Note

PERCENTILE_HIGH: 1.0

PERCENTILE_MID: 0.99999

PERCENTILE_LOW: 0.9999

Recommended Setting

PERCENTILE_MID

Table 18 start_ratio, end_ratio, and step_ratio

Name

See the Description.

Type

float

Value Range

end_ratio > start_ratio > 0 and step_ratio > 0 (If the input range is beyond the range that can be represented by the float type, the quantization precision of the model cannot be guaranteed.)

Description
  • start_ratio indicates the search start.
  • end_ratio indicates the search end.
  • step_ratio indicates the search step.

Note

After d_max and d_min are found in the algorithm, the ranges taken before and after d_max and d_min are determined based on start_ratio and end_ratio. Then, the step incremented each time is determined by step_ratio. For example:

If d_max = 100, start_ratio = 0.8, end_ratio = 1.2, and step_ratio = 0.01, the defined search range before and after d_max are 80 (100 x 0.8) and 120 (100 x 1.2) at a step of 1 (100 x 0.01). That is, there are 41 d_max search values in total.

Recommended Setting

The following two sets of settings are recommended:

  • start_ratio = 0.7, end_ratio = 1.3, step_ratio = 0.01
  • start_ratio = 0.3, end_ratio = 1.7, step_ratio = 0.01

Table 19 batch_count

Name

Number of batches of images processed in the quantization calibration set

Type

uint32

Value Range

[0, UINT32_MAX)

Description

Determines the number of batches of images processed in the quantization calibration set.

Note

If this parameter is not set or set to 0, all images in the calibration set are used. If the value is greater than 0, the smaller one in the total number of images in the calibration set path and the value of this parameter is used as the actual number of images in the calibration set. It is recommended that the number of images in the calibration set do not exceed 500.

Recommended Setting

None

Table 20 exclude_op

Name

Quantization operator blacklist

Type

string

Value Range

Operator name

Description

The operator indicated by this parameter is not quantified.

Note
  1. Only Conv, FC, and DepthWiseConv operators are supported.
  2. To blacklist multiple operators, multiple exclude_op parameters need to be set. Each parameter occupies a line. For example:

    exclude_op:'aaa'

    exclude_op:'bbb'

Recommended Setting

None

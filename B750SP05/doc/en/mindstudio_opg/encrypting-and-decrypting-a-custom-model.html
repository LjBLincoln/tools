Encrypting and Decrypting a Custom Model

Encrypting and Decrypting a Custom Model

By encrypting models, model developers can control the use of models. They can use the encryption function to obtain the encrypted offline model and key. Only users with the encrypted model and key can use the model.

An encrypted model can run only in a Mind project whose Target is set to ASIC or Atlas DK.

Encrypting a Model

  1. Select a project, choose Tools > Convert Model, or click of My Models on the tool tab page.

  2. Expand Optional Options and turn Encryption on, as shown in Figure 1.

    Figure 1 Turning Encryption on

  3. Enter the password. The encryption system uses this password to generate a unique key, which is used as an input for encryption.

  4. If you have entered a password for the workspace, the previous password is used by default. You can also choose to a new password, as shown in Figure 2.

    Figure 2 Use Last Password

  5. Upload the ISV hardware key, ISV certificate, and ISV private key, as shown in Figure 3.

    Figure 3 Uploading the hardware key, certificate, and private key

  6. After the model conversion is complete, the encrypted model file and PASSCODE file for decrypting the model are generated.

    After the model conversion is successful, a dialog box is displayed, indicating that the conversion is successful, as shown in Figure 4. Offline Model Path indicates the path of the encrypted file.

    Figure 4 Successful conversion dialog box

    Log in to the server where the Mind Studio tool is located. You can view the encrypted model file and passcode file in the path indicated by Offline Model Path.

Decrypting a Model

  1. Add an encrypted offline model to the draggable components.

  2. Drag the offline model to the orchestration canvas and go to the property tab page, as shown in Figure 5.

    Figure 5 Model property tab page

  3. Set Decryption to on, select the passcode file, and decrypt the model.

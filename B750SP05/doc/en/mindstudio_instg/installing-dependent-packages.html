Installing Dependent Packages

Installing Dependent Packages

Prerequisites

  • If the common user has the sudo permission, begin installation directly.

  • If the common user does not have the sudo permission, perform the configuration as follows.

    1. Run the following commands to open the /etc/sudoers file:

      su root

      chmod u+w /etc/sudoers

      vi /etc/sudoers

    2. Add the following content to the # User privilege specification file:

      username ALL=（ALL:ALL） NOPASSWD:SETENV:/usr/bin/apt-get

      username indicates the name of the common user who executes the installation script.

    3. Run :wq! to save the file.

    4. Run the following command to remove the write permission on the /etc/sudoers file:

      chmod u-w /etc/sudoers

      NOTE:
      Ensure that the last line of the /etc/sudoers file is #includedir /etc/sudoers.d. If the information does not exist, add it manually.

Procedure

Switch to the common user and perform the following operations:

  1. Run the following command to install the dependency （You are advised to copy the following commands to Word or Notepad and copy them to the server for execution） :

    sudo apt-get install gcc g++ cmake curl libboost-all-dev libatlas-base-dev unzip haveged python-skimage python3-skimage python-pip python3-pip liblmdb-dev libhdf5-serial-dev libsnappy-dev libleveldb-dev make graphviz autoconf libxml2-dev libxml2 libzip-dev libssl-dev sqlite3 python

    • During the python-skimage installation, software python, python-cycler, python-decorator, python-matplotlib, python-numpy, python-pil, python-pyparsing, python-dateutil, python-tz, and python-six will be introduced. The software will be checked later.

      • If a message is displayed indicating that the python-skimage software is not installed, run the following command to install the software:

        sudo apt-get install xxx

        NOTE:
        Replace xxx with the software introduced during the python-skimage installation.

      • If most of the python-skimage software is missing, run the following commands:

        sudo apt-get remove python-skimage

        sudo apt-get install python-skimage

    • During the python3-skimage installation, software python3, python3-cycler, python3-decorator, python3-matplotlib, python3-numpy, python3-pil, python3-pyparsing, python3-dateutil, python3-tz, and python3-six will be introduced. The software will be checked later.

      • If a message is displayed indicating that the python-skimage software is not installed, run the following command to install the software:

        sudo apt-get install xxx

        NOTE:
        Replace xxx with the software introduced during the python3-skimage installation.

      • If most of the python-skimage software is missing, run the following commands:

        sudo apt-get remove python3-skimage

        sudo apt-get install python3-skimage

    NOTE:
    Run make graphviz autoconf libxml2-dev libxml2 libzip-dev libssl-dev sqlite3 python to install the dependency required for the installation of profiling.

  2. Install the JDK.

    1. Run the sudo apt-get install -y openjdk-8-jdk command.

      NOTE:

      • If the JDK is installed on the local host, run the java -version command to check the version number. If the version number is earlier than 1.8.0_171, uninstall the JDK, download the sources again by referring to section 3.1.2, and reinstall the JDK.
      • If the JDK is installed by using the source described in Setting the Source and the commands described in 2.a, run the java -version command to check the version number. If the version number is earlier than 1.8.0_171, run the sudo apt-get update command to update the source.
      • If the source version is still earlier than 1.8.0_171 after performing either of the preceding step, download the jce_policy-8.zip file from the Oracle official website and replace the local_policy.jar and US_export_policy.jar files in the package to their counterparts in %JAVA_HOME%\jre\lib\security （%JAVA_HOME% is an environment variable, which can be viewed in the .bashrc file）.

    2. Set the environment variable.

      The installation and running of Mind Studio depend on the JAVA_HOME environment variable. If the environment variable is not set, the Mind Studio installation script exits the installation. Set JAVA_HOME as follows:

      1. Run the vi ~/.bashrc command in any directory to open the .bashrc file.

      2. Add the following content to the end of the file:

        export JAVA_HOME=/usr/lib/jvm/java-8-openjdk-amd64 
export PATH=$JAVA_HOME/bin:$PATH

        NOTE:
        JAVA_HOME indicates the JDK installation directory. If the JDK has been set, change it as required. If the JDK is installed according to the preceding steps, you do not need to change the installation directory.

      3. Run the :wq! command, save the file, and exit.

      4. Run the source ~/.bashrc command for the environment variable to take effect:

      5. Run the echo $JAVA_HOME command to check the environment variable setting. The following information is displayed:

        /usr/lib/jvm/java-8-openjdk-amd64

      6. Run the which jconsole command to check the JDK installation.

        If the following information is displayed, the installation is successful. Otherwise, the JDK installation fails.

        /usr/lib/jvm/java-8-openjdk-amd64/bin/jconsole

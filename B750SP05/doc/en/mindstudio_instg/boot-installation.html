Boot Installation

Boot Installation

Boot installation guides the user in setting parameters and installing Mind Studio and DDK. You are advised to use this installation mode preferentially.

Procedure

  1. Decompress the installation package.

    Upload the file mini_mind_studio_*.rar obtained as described in Software Package Preparation to the home directory of the Linux common user where Mind Studio is to be installed.

    Run the unzip mini_mind_studio_*.rar command to decompress the package.

  2. Run the installation script ./install.sh or bash install.sh.

  3. Follow the instructions to perform the settings.

    1. If the common user who installs Mind Studio does not have the sudo permission, the following message is displayed and the installation exists.

      Please check if add_sudo.sh, del_sudo.sh exists and execute the add_sudo.sh script with root privileges

      This message indicates that the common user does not have the permission to perform the installation. You need to switch to the root user to add the permission:

      su root

      ./add_sudo.sh username

      Then, switch to the common user and run the ./install.sh command again.

      NOTE:
      If the installation fails, re-add the permission as the root user.

    2. If the common user who installs Mind Studio has the sudo permission, the following information is displayed:

      Successfully add permission!

    3. Specify whether to install the Linux kernel patch.

      [INFO] Please make sure that the Linux Kernel Version is higher than 4.18 or apply this patch https://bugzilla.kernel.org/attachment.cgi?id=277305 when Linux Kernel Version below 4.18, otherwise maybe stuck when converting model using omg with TE plugin. Continue? [Y/N]: （Enter Y or y and press Enter to reject the installation of the Linux kernel patch, or enter N or n and press Enter to install the Linux kernel patch before installing Mind Studio.）

    4. Specify whether to use the default settings for installation.

      [INFO] Installation will run automatically with DEFAULT config, [Y/N]: （Enter Y or y and press Enter. The default settings will be used for installation. Otherwise, you need to set the parameters yourself.）

    5. Configure the IP address used for starting Mind Studio. Either of the following situations may occur.

      [INFO] Your ip address is xxx.xxx.xxx.xxx

      • [INFO] Press ENTER to continue or input a NEW ip address: （The IP address of the eth0 Ethernet adapter is obtained. Press Enter to continue or enter a custom IP address）
      • [INFO] Get ip address fail, please input your ip address: （The IP address of the eth0 Ethernet adapter is not obtained. Enter a custom IP address.）

    6. Select the backup path for saving data when the Mind Studio is uninstalled.

      [INFO] Please input backup path: （default is /home/username/wsbackup）: （Input a backup path or press Enter to use the default setting.）

    7. Specify whether to load data from the backup path. The following information is displayed only when the default path is selected in Step 3.f and /home/username/wsbackup is not empty:

      [INFO] Do you want to load data from /home/username/wsbackup ? [Y/N]: （Enter Y or y and press Enter to load data from /home/username/wsbackup, or enter N or n and press Enter to reject the loading.）

    8. Set the Mind Studio listening port. In Step 3.e, if Y or y is entered and Enter is pressed to use the default configuration, this step is skipped.

      [INFO] Please input you port （default is 8888）: （Input a port number or press Enter to use the default setting.）

    9. Set tool_path for saving tools during Mind Studio is running. In Step 3.e, if Y or y is entered and Enter is pressed to use the default configuration, this step is skipped.

      [INFO] Please input tool path （default is /home/username/tools）: （Input a tool path or press Enter to use the default setting.）

      1. Set profiler_port （Profiling listening port）. In Step 3.e, if Y or y is entered and Enter is pressed to use the default configuration, this step is skipped.

      [INFO] Please input you profiler port （default is 8099）: （Input a port number or press Enter to use the default setting.）

    11. Set apache_user （user name of the Apache service running on the background of Profiling）. In Step 3.e, if Y or y is entered and Enter is pressed to use the default configuration, this step is skipped.

      [INFO] Please input you apache_user （default is msvpUser）: （Input the Apache user name or press Enter to use the default configuration.）

    12. Specify whether to back up the tools installation directory. （If the installation directory is not empty, this message is displayed.）

      [INFO] /home/username/tools is not empty. Files in the directory will be cleared during installation. Are you sure to back up them?[Y/N]: （Enter Y or y and press Enter to exit the installation. Back up the installation directory, and then run the /.install.sh script to perform the installation again. Enter N or n and press Enter to delete the contents in /home/username/tools and continue to install Mind Studio.）

    13. Check whether the local Python software needs to be replaced with a specified version.

      pip: numpy=1.14.3 decorator=4.0.10 Pillow=5.2.0 gevent=1.3.6 easydict=1.9 pyyaml=3.13

      pip3: numpy=1.14.3 decorator=4.0.10 Pillow=5.2.0 matplotlib=2.1.0

      [INFO] The following python packages need to be replaced, whether to overwrite the installation?[Y/N]: （Enter Y or y and press Enter to confirm the version replacement and continue the installation, or enter N or n and press Enter to reject the version replacement and exit the installation.）

      NOTE:

      • The common user must have the permission to create the tool path and backup path.
      • For details about parameters such as the IP address, port number, tool path, and backup path, see 2.

  4. Install Mind Studio and the DDK.

    Mind Studio is installed in the tools directory where the install.sh script is stored.

    If the installation fails, see FAQs.

    NOTE:
    After the installation is complete, the del_sudo.sh script is automatically executed to revoke the permission of the common user, no matter the installation succeeds or fails.

Uninstalling Mind Studio

Uninstalling Mind Studio

Log in to Mind Studio as the installation user and run the ./uninstall.sh command in the ~/tools/bin directory on Linux to uninstall the Mind Studio.

During the uninstallation, perform the following operations:

  1. If the common user who uninstalls Mind Studio does not have the sudo permission, the following message is displayed and the uninstallation exists.

    Please check if add_sudo.sh, del_sudo.sh exists and execute the add_sudo.sh script with root privileges

    This message indicates that the common user does not have the permission to perform the installation. You need to switch to the root user to add the permission:

    su root

    ./add_sudo.sh username

    Then, switch to the common user and run the ./uninstall.sh command again.

    NOTE:
    If the uninstallation fails, add the permission as the root user.

  2. If the common user who uninstalls Mind Studio has the sudo permission, the following information is displayed:

    Successfully add permission!

  3. Specify whether to back up the user data.

    [WARNING] Do you need to backup for user data （include projects my-datasets my-model caffe-model mongodb profiling） ? [Y/N]

    Enter Y or y and press Enter to back up the user data and 3.a, or enter N or n and press Enter to skip the backup and enter 3.b.）

    • If user data is backed up, the following message is displayed:

      [WARNING] Directory /home/username/wsbackup is not empty, please make your choice! [Y:（Continue to overwrite backup）/N:（Change a directory）]:

      Enter Y or y and press Enter to overwrite the backup, or enter N or n and press Enter to change the backup path.

    • If the user data is not backed up, the system prompts whether to delete the user data.

      [WARNING] Are you sure to remove user data （include projects my-datasets my-model caffe-model mongodb profiling） ? [Y/N]:

      Enter Y or y and press Enter to delete the user data or enter N or n and press Enter to retain the user data.

  4. Uninstall MongoDB.

  5. Stop the HiAI_CCE-Profiler service.
  6. Stop the web service.
  7. Back up user data to the backup directory, including project, my-model, and my-datasets.

NOTE:
After the uninstallation, if you want to decompress the installation package again or decompress the installation package of a later version, delete the del_sudo.sh and profiling_sudo.sh scripts from the original package, switch to the root user, and run the following commands:
su root
chattr -i del_sudo.sh profiling_sudo.sh
rm -rf del_sudo.sh profiling_sudo.sh

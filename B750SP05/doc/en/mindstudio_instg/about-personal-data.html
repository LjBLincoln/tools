About Personal Data

About Personal Data

Personal Data

Usage Scenario

When logging in to Mind Studio, the user needs to enter the user name and password. The IP address of the host is collected.

The data set imported by the user is used when the model is used for performing inference. The data set may contain personal privacy data.

Collected Items

IP address, user name, password, and data set

Collection Source and Method

User names and passwords are collected during user registration, IP addresses are collected during login and logout, and data sets are uploaded by users.

Purpose and Security Protection Measure
  1. User names and passwords are used as credentials for user login authentication. IP addresses are used to identify the client for user operations during audit, and data sets are used by users to perform inference on data. In addition, when you use Mind Studio, the server where Mind Studio is installed records your latest account login and operation logs. The operation logs include the IP address and user name for security audit.
  2. Data is transmitted under HTTPS-based encryption.
  3. User names and passwords are encrypted using PBKDF2WithHmacSHA256 and stored on the local Mind Studio server. IP addresses are recorded in logs in plain text. Log files are controlled by the permission (600) and data sets are controlled by the permission (600).

Data Retention Period and Policy
  • The system forcibly changes the user name and password every three months by default.
  • The log data collected by Mind Studio is recorded on the host where Mind Studio is deployed. Therefore, the log can be manually deleted and cleared by the user. The maximum size of an operation log file is 20 MB. If the maximum size is exceeded, a new operation log file is generated. The maximum number of operation log files is 30. If the maximum number is exceeded, the generated operation log files overwrite the earliest operation log files.
  • Due to repeated inference, data sets are retained on the Mind Studio server during inference.

Clearance Method
  • Users can delete personal data (such as the user names, passwords, IP addresses, and imported data sets) by uninstalling Mind Studio or on the GUI.
  • Users can delete personal data by using profiling on the GUI.
  • Due to repeated inference, data sets are stored in the ~/HIAI_DATANDMODELSET/workspace_mind_studio directory of the server on the host during inference and need to be cleared by users.
